# 方案推送模板

## 概述

方案推送是 Skill 开发的第三步，目标是主推一个方案，附带 2-3 个备选，让用户选择。

## 搜索完成后暂停机制

### 强制暂停规则

**核心原则：搜索完成后，系统必须立即暂停，不得自动继续。**

搜索完成后，系统必须执行以下流程：

1. **暂停执行**
   - 搜索完成后，立即停止后续操作
   - 不得自动进入方案生成或实施阶段
   - 等待用户明确指示

2. **展示搜索结果**
   - 显示搜索到的关键信息
   - 列出找到的相关资源、工具、项目
   - 标注搜索结果的来源和可靠性

3. **等待用户确认**
   - 必须等待用户选择或修改
   - 用户确认后才继续下一步
   - 不得假设用户意图

### 搜索结果展示格式

```markdown
## 🔍 搜索结果摘要

**搜索关键词：** [用户输入的关键词]

**找到的相关资源：**
1. [资源 1 名称] - [简要说明] - [来源]
2. [资源 2 名称] - [简要说明] - [来源]
3. [资源 3 名称] - [简要说明] - [来源]

**推荐方向：**
基于搜索结果，建议从以下方向考虑：
- [方向 1]
- [方向 2]
- [方向 3]

**⚠️ 请确认后继续：**
请选择：
1. 继续基于搜索结果生成方案
2. 调整搜索关键词重新搜索
3. 补充更多需求信息
4. 直接指定方向
```

### 暂停检查点

系统在以下节点必须暂停：

| 检查点 | 暂停时机 | 展示内容 |
|--------|----------|----------|
| 搜索完成 | 搜索结果返回后 | 搜索结果摘要、推荐方向 |
| 方案生成 | 推荐方案生成后 | 方案详情、备选方案、对比表 |
| 方案修改 | 用户提出修改后 | 修改后的方案、变化对比 |
| 最终确认 | 方案确定后 | 最终方案、实施计划 |

## 推荐方案生成

### 生成要求

1. **基于搜索结果和用户需求**
   - 分析搜索结果
   - 结合用户需求
   - 选择最佳匹配的方案

2. **详细说明实现思路**
   - 列出实现步骤
   - 说明每个步骤的具体内容
   - 预估每个步骤的时间

3. **预估开发时间和难度**
   - 总时间估算
   - 难度评估（1-5 星）
   - 风险评估

### 输出格式

```markdown
## 🎯 推荐方案：[方案名称]

**推荐理由：** [一句话说明为什么推荐]

**实现思路：**
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

**技术栈：** [技术栈列表]
**预估时间：** [时间]
**难度：** ⭐⭐⭐⭐⭐
**风险：** [低/中/高]
```

## 备选方案生成

### 备选方案类型

1. **备选方案 1：轻量级**
   - 适用场景：快速验证想法
   - 特点：实现快，简单
   - 缺点：功能有限
   - 预估时间：较短

2. **备选方案 2：功能最全**
   - 适用场景：需要完整功能
   - 特点：功能全面
   - 缺点：开发时间长
   - 预估时间：较长

3. **备选方案 3：创新方案**
   - 适用场景：想尝试新技术
   - 特点：技术先进
   - 缺点：风险较高
   - 预估时间：不确定

### 输出格式

```markdown
## 🔄 备选方案

### 备选方案 1：轻量级
**适用场景：** 快速验证想法
**优点：** 实现快，简单

**预估时间：** [时间]

### 备选方案 2：功能最全

**优点：** 功能全面
**缺点：** 开发时间长
**预估时间：** [时间]

### 备选方案 3：创新方案

**优点：** 技术先进
**缺点：** 风险较高
**预估时间：** [时间]
```

## 方案对比

### 对比维度

1. **技术栈**
   - 使用了哪些技术
   - 技术是否先进
   - 技术是否稳定

2. **优点**
   - 实现了哪些功能
   - 有哪些优势

3. **缺点**
   - 缺少哪些功能
   - 有哪些劣势

4. **预估时间**
   - 开发时间估算
   - 测试时间估算
   - 部署时间估算

5. **难度**
   - 技术难度（1-5 星）
   - 实现难度（1-5 星）
   - 测试难度（1-5 星）

6. **推荐度**
   - 综合评分（1-5 星）
   - 推荐理由

## 方案修改机制

### 修改触发条件

用户可以基于以下原因提出方案修改：

1. **搜索结果不满意**
   - 搜索结果不相关
   - 搜索结果过时
   - 需要更多搜索范围

2. **推荐方案不符合预期**
   - 技术栈不合适
   - 实现思路不认同
   - 时间/难度不合适

3. **需求变更**
   - 用户补充了新需求
   - 需求范围发生变化
   - 优先级调整

### 修改类型

#### 1. 修改推荐方案

用户可以要求修改推荐方案的任意部分：

```markdown
## ✏️ 修改推荐方案

**原方案：** [原方案名称]
**修改内容：**
- [修改项 1]：[原内容] → [新内容]
- [修改项 2]：[原内容] → [新内容]

**修改后方案：**
[修改后的完整方案]
```

#### 2. 重新搜索

用户可以要求重新搜索：

```markdown
## 🔄 重新搜索

**原搜索关键词：** [原关键词]
**新搜索关键词：** [新关键词]
**搜索范围调整：** [调整说明]

系统将基于新关键词重新搜索并生成方案。
```

#### 3. 自定义方案

用户可以直接提供自己的方案：

```markdown
## 🛠️ 自定义方案

**用户方案：**
[用户提供的方案描述]

**系统评估：**
- 技术可行性：[评估]
- 时间估算：[估算]
- 难度评估：[评估]
- 风险提示：[提示]

**建议调整：**
[系统基于专业角度的建议]
```

### 修改流程

```
用户提出修改
    ↓
系统理解修改意图
    ↓
确认修改内容
    ↓
执行修改
    ↓
展示修改结果
    ↓
再次暂停等待确认
    ↓
用户确认或继续修改
```

### 修改限制

1. **最大修改次数：** 建议不超过 3 轮修改
2. **修改范围：** 可以修改方案的任意部分
3. **必须确认：** 每次修改后必须等待用户确认

## 方案确认流程

### 确认机制

**核心原则：方案展示后，必须等待用户明确选择，不得自动继续。**

系统展示方案后，必须执行以下确认流程：

1. **展示完整方案**
   - 推荐方案详情
   - 所有备选方案
   - 方案对比表
   - 搜索结果摘要

2. **等待用户选择**
   - 必须等待用户明确输入
   - 不得假设用户选择
   - 不得自动进入下一步

3. **处理用户选择**
   - 根据用户选择执行对应操作
   - 执行后再次展示结果
   - 再次等待确认

### 用户选择选项

```markdown
## 🤔 请选择方案

基于搜索结果和分析，推荐以下方案。请明确选择：

**方案选择：**
1. ✅ 使用推荐方案 - [推荐方案名称]
2. 🔄 使用备选方案 1 - [备选方案 1 名称]
3. 🔄 使用备选方案 2 - [备选方案 2 名称]
4. 🔄 使用备选方案 3 - [备选方案 3 名称]

**方案调整：**
5. ✏️ 修改方案 - 告诉我你想修改哪些部分
6. 🔍 重新搜索 - 调整搜索关键词或范围
7. 🛠️ 自定义方案 - 提供你自己的方案想法

**其他操作：**
8. ❓ 查看更多细节 - 深入了解某个方案
9. ⏸️ 暂停 - 稍后再继续

请输入选项编号或描述你的需求：
```
